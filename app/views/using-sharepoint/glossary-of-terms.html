Definitions of request statuses in SharePoint

1. New request

A new request has been submitted to an information manager through the ‘Request support as a voluntary or community organisation’ online service. It needs to be passed on to a liaison lead for triage.

2. Review

A liaison Lead needs to check locally before further action is taken on the request.

3. Exploring local

The request is being handled locally.

4. Escalated to MAC

The request cannot be met locally and needs brokering at next Multiagency Cell (MAC) meeting.

5. More information

The request requires more information from the requesting organisation before a broker can be assigned.

6. Broker found

A broker has been assigned to the request at the Multiagency Cell meeting. The information manager should transfer the request to that broker.

7. Escalate to SCG

No broker was able to supply support. The request should be pushed to a national level, through the Strategic Coordination Group (SCG).

8. SCG response

The SCG has reported back on the request. The action they have taken is in the SCG response comment box.

9. Closed

Request is finished, no further action will be taken on it.
